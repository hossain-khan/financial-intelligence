# Qualification Matrix

## Purpose

Turn the browser, accessibility, and performance goals into repeatable release gates on
representative synthetic financial workloads. This complements [Quality baseline](17-QUALITY-BASELINE.md)
with the workload tiers, performance budgets, capability matrix, and manual release-evidence
template introduced in issue #29 (see [ADR-016](adr/ADR-016-Qualification-Matrix.md)). No benchmark
claim is published without reproducible environment metadata.

## Workload tiers

Deterministic, reconciled, synthetic workloads are generated by `@financial-intelligence/qualification`
(`generateWorkload`). Each records a manifest (generator version, seed, config, record counts, byte
size, content digest, expected decimal-safe totals) so a data change cannot masquerade as a
performance change. Nothing derives from a real statement.

| Tier | Transactions | When it runs | Owner |
| --- | --- | --- | --- |
| `smoke1k` | 1,000 | Every PR (informational CI job) | CI |
| `smoke10k` | 10,000 | On demand / pre-merge for import-heavy changes | Author |
| `release50k` | 50,000 | Release candidate, scheduled | Release owner |
| 250k bound | 250,000 | Documented NFR ceiling; raise `transactionCount`, run on a physical device | Release owner |

Run the PR smoke locally:

```bash
PERF_ROWS=1000 PERF_ITERATIONS=5 pnpm exec playwright test --project=perf
node scripts/check-perf-result.mjs perf-results/qualification-1000.json
```

## Performance budgets

Budgets map to the non-functional requirements. CI measures them as a **regression smoke signal**;
release thresholds are confirmed on pinned physical reference devices. End-to-end UI measures
(measured through the real import/ledger/dashboard UI) legitimately exceed the raw-operation NFR
figures — interpret them with the recorded environment profile.

| Metric id | NFR | Budget | Direction |
| --- | --- | --- | --- |
| `app-interactive` | NFR-020 | ≤ 2,000 ms (warm) | max |
| `ledger.render` / `ledger.filter` | NFR-021 | ≤ 200 ms | max |
| `dashboard.query` | NFR-022 | ≤ 1,000 ms | max |
| `csv.throughput` | NFR-023 | ≥ 10,000 rows/s | min |
| main-thread long tasks | NFR-024 | < 50 ms | max |
| `ledger.dom-rows` | NFR-025 | bounded (≤ 200; ledger paginates at 50) | max |

## Result schema and comparison

Results follow the versioned `PerfResult` schema (`packages/qualification/src/result-schema.ts`):
commit/dirty/node/pnpm/browser/OS/hardware, workload digest+label, cold/warm mode, iterations, raw
samples, median, p95, nullable memory, threshold, threshold direction, pass/fail. `compareResults`
only compares **matching environment profiles** — a different browser, OS, hardware, or workload
digest is a new baseline, never an automatic regression. Every artifact passes
`assertNoSensitiveContent`, which rejects descriptions, account labels, monetary amounts, and any
free text; artifacts carry only timings, counts, digests, and identifiers.

## Supported browser and capability matrix

Core no-AI workflows **do not require WebGPU**. WebGPU/WASM/worker/persistence/install/memory
capability is recorded as informational input for Phase 4, never a core pass condition.

| Browser family | PR evidence | Release evidence |
| --- | --- | --- |
| Chromium (Chrome/Edge) | Playwright Chromium project + perf smoke | Current Chrome and Edge stable |
| Firefox | Playwright Firefox project | Current Firefox stable |
| WebKit (Safari) | Playwright WebKit project | Current macOS Safari + one current iOS/iPadOS device |

| Capability | Core requirement | Recorded for |
| --- | --- | --- |
| Service worker + install | Required for offline/install journeys; degrade with guidance when absent | PWA lifecycle |
| Storage persistence | Not required; denial is a supported state | Storage settings |
| WebAssembly | Required for Argon2id backup KDF | Backup/restore |
| Web Workers | Required for parser/crypto isolation | Import, backup |
| WebGPU | **Not required for core**; informational only | Phase 4 AI |

### Unsupported / degraded states to verify

- Private-browsing quota limits and storage eviction — usable, with clear messaging.
- No install prompt (non-Chromium) — accurate platform instructions instead of a false claim.
- Persistence denied — data still stored; warned it may be evicted.
- WebGPU absent — core ledger, import, dashboards, and backup fully usable.

## Manual release-evidence template

Complete for each release candidate; record and store with release evidence.

- [ ] Environment: browser, version, OS, device, build commit, reviewer, date.
- [ ] Chrome/Edge, Firefox, macOS Safari, one iOS/iPadOS device: core import → ledger → dashboard →
      backup/restore completed.
- [ ] Installed-PWA offline reload in Firefox, macOS Safari, and iOS/iPadOS Safari.
- [ ] Screen reader: landmarks, headings, form labels, status/error announcements, ledger and chart
      table semantics.
- [ ] Forced-colors / high-contrast: focus and status not by color alone.
- [ ] 200% zoom and 320 CSS pixel reflow: no clipping, no two-dimensional scroll, controls usable.
- [ ] Touch-target size on a real device.
- [ ] macOS Safari full-keyboard-access: skip link and all controls reachable.
- [ ] `release50k` perf run on a pinned physical reference device; attach the result artifact.
